# CLAUDE.md

## プロジェクト名
白詠舎（Byakueisha）Webサイト

## このプロジェクトの目的
このプロジェクトは、架空の陶芸・器ブランド「白詠舎」のポートフォリオ用Webサイトを制作するものです。  
完成物は、制作者のポートフォリオに掲載するサンプルサイトとして使用します。

## 最重要ルール
実装やデザイン調整を行う前に、必ず `DESIGN.md` を参照してください。  
このサイトは、派手なLPやECサイトではなく、静かで端正な工房・器ブランドサイトです。

## サイト構成
以下の3ページ構成で制作してください。

- `/` : Topページ
- `/about` : Aboutページ
- `/utsuwa` : Utsuwaページ

Contactは独立ページにせず、各ページ下部の導線またはメールリンクで対応して構いません。

## 写真素材の扱い

写真は `public/images` 配下に配置される想定です。  
画像ファイル名は、英小文字とハイフンで管理してください。

例：
```txt
public/images/hero-ceramics.jpg
public/images/about-entrance.jpg
public/images/utsuwa-bowl.jpg
public/images/studio-kiln.jpg
```

### 写真の使い方
- TopのHeroは、器の静物写真または工房の空気感が伝わる写真を使用する
- 人が手で器を持つ写真は、Heroでは基本使わない
- 手元写真は、Aboutまたは制作風景の補助素材として使う
- 外観写真はAboutやAccess導線で使用する
- 商品写真はUtsuwaページで使用する

## 実装方針

### 基本
- まずはシンプルに構築する
- 保守しやすい構成にする
- CSSは読みやすく整理する
- コンポーネントを分けすぎず、初心者にも追いやすい構造にする
- ただし共通ヘッダー、フッター、ボタン、カードなどは再利用できる形にする

### 推奨構成例
```txt
app/
  page.tsx
  about/
    page.tsx
  utsuwa/
    page.tsx
components/
  Header.tsx
  Footer.tsx
  Button.tsx
  SectionTitle.tsx
  UtsuwaCard.tsx
public/
  images/
styles/
  globals.css
```

Next.js App Routerを使う場合は、上記を基本にしてください。  
もし別構成にする場合も、できるだけシンプルで見通しの良い構成にしてください。

## コンテンツ方針

### Topページ
Topページは世界観を見せる入口です。  
文章は短めにし、写真と余白で印象を作ってください。

入れる内容：
- Header
- Hero
- Concept
- About導線
- Utsuwa導線
- 制作風景
- 工房案内 / Access
- Contact導線
- Footer

### Aboutページ
白詠舎の背景やものづくりの考えを伝えるページです。  
外観写真、工房写真、手元写真を使って、静かに深みを出してください。

入れる内容：
- ページタイトル
- 工房の入口・外観写真
- 白詠舎について
- ものづくりの考え
- 素材・土について
- 作り手・工房風景
- 工房情報
- Contact導線
- Footer

### Utsuwaページ
器そのものを見せる作品紹介ページです。  
ECサイトではないので、カートや購入ボタンは不要です。

入れる内容：
- ページタイトル
- 導入文
- 器一覧
- 取扱・展示についての案内
- Contact導線
- Footer

## コピー


## UIルール

### ボタン
- 角丸にしすぎない
- 細い枠線または控えめな塗り
- ホバーは薄く変化する程度
- 文言は短くする

例：
```txt
詳しく見る
うつわ一覧を見る
お問い合わせ
```

### 余白
- セクション間は広め
- 写真と文章の間も広め
- 詰め込み禁止

### 罫線
- 必要な場合のみ細い線を使う
- 装飾目的で多用しない

### アニメーション
- 原則なし
- 入れる場合はフェードイン程度

## 実装時に勝手に追加しないもの

## 作業の進め方

1. まず `DESIGN.md` を読み、サイトの方向性を把握する
2. 3ページ構成を確認する
3. 既存画像素材があれば、適切な場所に配置する
4. Topページから実装する
5. Aboutページを実装する
6. Utsuwaページを実装する
7. 全体の余白、文字サイズ、写真の見え方を調整する
8. 最後にスマホ表示を確認する

## ユーザーに確認した方がよいこと
判断に迷った場合、勝手に大きく変更せず、以下は確認してください。

- 写真の採用可否
- ページ構成の変更
- ブランド名やコピーの変更
- Contactページを独立させるかどうか
- CMS導入の有無
- 価格表示を入れるかどうか
